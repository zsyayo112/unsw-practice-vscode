# UNSW Practice 项目规划

---

## 🎯 项目目标

**一句话定位：** UNSW CS 学生的一站式学习平台，从练题工具起步，逐步成为选课、笔记、社区的信息中心，核心商业目标是为补课机构持续引流。

**成功标准：**
- 阶段一结束：COMP9021 这个 term 50% 的同学知道这个插件
- 阶段二结束：平台注册用户超过 300 人
- 阶段三结束：成为 UNSW CS 中文学生群体的"第一站"

## 📦 阶段一：VSCode 插件 MVP

**时间目标：6 周**
**核心指标：** 让 COMP9021 练题体验比任何地方都好

### 为什么从插件开始

- 门槛足够高，竞争对手不会轻易复制
- 学生写代码本来就在 VSCode 里，零切换成本
- 题目 100% 对应 UNSW 课程，LeetCode 无法替代

### 功能范围

**做：**
- 侧边栏题目列表（按难度/Topic 分组）
- 做题面板（Monaco 编辑器 + 题目描述）
- 代码运行和测试结果展示
- 死循环保护（3 秒超时）
- Mock 数据支持（先跑通 UI）

**不做：**
- 用户登录（MVP 不需要）
- 云端进度同步（本地 localStorage 足够）
- 多语言支持（只支持 Python）
- 排行榜、讨论区

### 6 周开发计划

```
Week 1：环境搭建 + 项目脚手架
  - WSL2 + Node.js + Claude Code 环境配置
  - yo code 生成项目骨架
  - package.json 配置完成
  - F5 能跑起来 Hello World

Week 2：侧边栏题目列表
  - ProblemTreeProvider 实现
  - Mock 数据 3-5 道题
  - 按难度分组显示
  - 点击题目有响应

Week 3：做题面板 WebView
  - 左右分栏布局
  - 题目描述渲染（Markdown）
  - Monaco 编辑器初始化
  - 代码自动保存（localStorage）

Week 4：代码运行
  - 接入 Piston API
  - 测试用例结果展示
  - 超时保护（3 秒）
  - 错误信息友好展示

Week 5：联调后端 API
  - 后端 API 搭建（Next.js API Routes）
  - 替换 Mock 数据为真实题库
  - 题目数据库入库（现有 py 文件解析）

Week 6：打磨 + 发布
  - UI 细节优化
  - 错误处理完善
  - 发布到 VSCode Marketplace
  - 写安装文档
```

### 技术栈

```
插件本体：TypeScript + esbuild
代码编辑：Monaco Editor（CDN）
代码执行：Piston API（免费沙箱）
后端 API：Next.js API Routes
数据库：Supabase（PostgreSQL）
部署：Vercel（后端）
```

### 推广策略

- 发到 COMP9021 的微信群
- 请机构现有学生试用，收集反馈
- 口碑为主，不做广告

## 🌐 阶段二：网页版 + 账号体系

**时间目标：阶段一结束后 8 周**
**核心指标：** 注册用户 300+，插件用户转化率 60%

### 为什么要做网页版

插件的天花板：
- 只服务一门课
- 没有用户身份，数据孤立
- 无法做社交功能

网页版解决：
- 账号体系 → 数据云端同步
- 可以做排行榜、进度展示
- 为阶段三社区功能打基础

### 功能范围

**做：**
- 用户注册/登录（Supabase Auth）
- 网页版练题界面（复用后端 API）
- 个人主页（做题进度、成就）
- 插件和网页进度同步
- 联系/预约页面（核心引流入口）

**不做：**
- Wiki 内容（阶段三）
- 课程评价（阶段三）
- 用户发帖（阶段三）

### 8 周开发计划

```
Week 1-2：Next.js 项目搭建 + 首页
  - 项目初始化（Next.js 14 App Router）
  - 首页设计和开发
  - 联系/预约页面

Week 3-4：认证体系
  - Supabase Auth 集成
  - 注册/登录页面
  - 插件 OAuth 登录流程

Week 5-6：练题模块网页版
  - 题库列表页
  - 做题界面（复用后端逻辑）
  - 进度云端同步

Week 7：个人主页
  - 做题统计
  - 成就系统
  - 班级排行榜

Week 8：打磨 + 上线
  - SEO 基础配置
  - 性能优化
  - 正式上线
```

  - 难度/工作量评分

```

         ↓
学生自然传播
         ↓
更多用户贡献内容
         ↓
内容质量提升
         ↓
更多学生来用
         ↓
机构咨询量增加
```

## 💰 商业逻辑

```
阶段一：
  插件免费 → 展示机构教学水平
  → 用户扫码加微信 / 预约试听

阶段二：
  网页免费 → 注册用户增长
  → 邮件/通知推送机构活动
  → 预约页面转化

阶段三：
  社区免费 → 平台品牌建立
  → 被动流量持续增长
  → 未来可考虑：课程资料付费、会员制
```

**核心原则：平台永远免费，机构服务付费。**
平台是鱼塘，机构是渔船，不能把鱼塘变成收费入场。

## ⚠️ 风险和对策

| 风险 | 可能性 | 对策 |
|------|--------|------|
| Piston API 限流 | 中 | 提前设计队列，用户量大了换方案 |
| UNSW 改课程内容 | 低 | 题库后台可以快速更新，不依赖插件版本 |
| 竞争对手跟进 | 低 | 先发优势 + 内容积累是护城河 |
| 一个人开发太慢 | 高 | MVP 功能严格控制，不做范围外的功能 |
| 用户不愿注册 | 中 | 插件先不强制登录，降低使用门槛 |
